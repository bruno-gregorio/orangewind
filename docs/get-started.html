Documentation

Orangewind is a Tailwind CSS v4 plugin that re-implements Canonical's Vanilla Framework as a set of component classes. You bring Tailwind v4; Orangewind layers its ow-* components on top and reuses Tailwind's native utilities rather than duplicating them.

Install

Orangewind is published on npm. It requires Tailwind CSS v4 as a peer dependency, so install both:

Terminal
bun add orangewind tailwindcss

Using npm, pnpm or yarn? Swap in your package manager's add command.

Import the stylesheet

Orangewind ships a single CSS entry point. Import Tailwind first, then Orangewind, from your global stylesheet — the same wiring this documentation site uses.

app.css
@import 'tailwindcss';
@import 'orangewind';

Syntax highlighting for code blocks (used across these docs) is opt-in via a separate stylesheet, so you only pay for it when you need it.

app.css (optional)
@import 'orangewind/prism.css';

Dark mode

Orangewind only flips its own CSS variables for dark mode — it never styles the document itself. By default Tailwind's dark variant follows the operating system's prefers-color-scheme, so the framework themes itself automatically with no toggle required. See the UI patterns guide for how to drive the theme from a class instead.

Next steps

With the stylesheet imported, read the Markup patterns guide to learn how to author Orangewind class markup, then browse the component reference.
