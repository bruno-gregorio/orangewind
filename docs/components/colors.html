Orangewind exposes its palette as --color-ow-* theme tokens, so every colour is available through standard Tailwind utilities such as bg-ow-brand, text-ow-positive and border-ow-information. The palette covers a neutral greyscale, the brand and accent hues, and a set of semantic colours.

Brand and accent

ow-brand is the primary identity colour and ow-accent is its complement. Apply them with any colour utility.

ow-brand
ow-accent
Markup
<div class="bg-ow-brand"></div>
<div class="bg-ow-accent"></div>

Semantic colours

Four semantic colours signal the meaning of an action or message: ow-positive, ow-negative, ow-caution and ow-information. Components reuse these tokens, so a custom element can match a notification or badge by sharing its colour.

Text colours

Body copy uses text-ow-text-default, with text-ow-text-muted for secondary text. The semantic tokens also work as text colours, for example text-ow-negative on an error message.

Backgrounds and borders

Surface tokens drive page and panel chrome. bg-ow-background-default and bg-ow-background-alt set page backgrounds, while border-ow-surface-border draws the subtle borders used throughout these docs.

Dark mode

Orangewind redefines its semantic tokens under prefers-color-scheme: dark, so background, text, border and surface colours flip automatically. Build with the semantic utilities — such as text-ow-text-default over bg-ow-background-default — rather than the raw greyscale, and a layout adapts to the user's theme with no extra work. The variables are remapped like this:

Adapts to the active colour scheme.
Always dark, for contrast.
CSS
:root {
  @variant dark {
    --color-ow-background-default: var(--color-ow-dark);
    --color-ow-text-default: var(--color-ow-x-light);
    --color-ow-border-default: color-mix(
      in srgb,
      var(--color-ow-x-light) 20%,
      transparent
    );
  }
}
