Documentation

Default

Package security, release cadence and support windows grouped in a quiet information surface.

Highlighted

A stronger surface for destinations where the whole card supports an interactive workflow.

Cards group related content on a contained surface. Reach for ow-card for a bordered card, ow-card-highlighted or ow-card-muted for shadowed surfaces, and ow-card-overlay for a card placed over imagery. Body copy can use ow-card-content to tune its trailing spacing.

Default

A bordered card for general grouped content.

Default

Package security, release cadence and support windows grouped in a quiet information surface.

Markup
<article class="ow-card">
  <h3 class="ow-heading-5">Default</h3>
  <p class="ow-card-content">
    Package security, release cadence and support windows grouped in a quiet
    information surface.
  </p>
</article>

Highlighted

ow-card-highlighted drops the border for a soft shadow, giving a stronger surface for interactive destinations.

Highlighted

A stronger surface for destinations where the whole card supports an interactive workflow.

Markup
<article class="ow-card-highlighted">
  <h3 class="ow-heading-5">Highlighted</h3>
  <p class="ow-card-content">
    A stronger surface for destinations where the whole card supports an
    interactive workflow.
  </p>
</article>

Muted

ow-card-muted is a low-emphasis variant for supporting information that still needs separation from neighbouring content.

Muted

Low-emphasis supporting information that still needs separation from neighbouring content.

Markup
<article class="ow-card-muted">
  <h3 class="ow-heading-5">Muted</h3>
  <p class="ow-card-content">
    Low-emphasis supporting information that still needs separation from
    neighbouring content.
  </p>
</article>

With a header

ow-card-header adds a divided header region. Pair an ow-card-thumbnail image with a title to introduce the card.

Raspberry Pi 5

Ubuntu Core supports compact device fleets with predictable updates and a small operational footprint.

Markup
<article class="ow-card">
  <div class="ow-card-header grid gap-3">
    <img class="ow-card-thumbnail" src="/path/to/logo.svg" alt="" />
    <h3 class="ow-heading-5">Raspberry Pi 5</h3>
  </div>
  <p class="ow-card-content">
    Ubuntu Core supports compact device fleets with predictable updates and a
    small operational footprint.
  </p>
</article>

With a full-bleed image

Remove the card padding with p-0, add an ow-card-image that fills the width, then place the text in an ow-card-inner region so it keeps its own padding.

The state of robotics

From social participation to environmental detection, robotics teams keep stretching what field hardware can do.

Markup
<article class="ow-card p-0">
  <img class="ow-card-image" src="/path/to/cover.jpg" alt="" width="496" height="278" />
  <div class="ow-card-inner grid gap-3">
    <h3 class="ow-heading-5">The state of robotics</h3>
    <p>
      From social participation to environmental detection, robotics teams keep
      stretching what field hardware can do.
    </p>
  </div>
</article>

Overlay

ow-card-overlay renders a translucent surface designed to sit on top of a background image, keeping its content legible over the imagery.
